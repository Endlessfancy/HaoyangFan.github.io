Research

Research under the instruction of Prof. Chen

 

Considering that HRRP contains rich identification information and efficient processing, target recognition based on HRRP has attracted extensive attention in the field of radar automatic target recognition (RATR). As for HRRP target recognition problem (HRRP itself has time dependence), the common methods includes Hiddien Markov Model (HMM) based methods, RNN, probability generation model generative model and so on. However, these methods also have limitations. All the kinetic models mentioned above are one-way models, which can only represent the forward information transmission in the HRRP sample and ignore the dependence of backward time. Therefore, none of these models can use all the available input information for the past and future of a particular time frame in the HRRP sequence characteristics. This is also the main purpose of the research, trying to combine the bidirectional nature of time.

a internship in Hkivision

 

The main task of the internship is to explore and test the existing living detection algorithm (Living detection refers to the task of distinguishing human and dummy, which may include photos, head mold, etc.) and try to break through the algorithm.

As depicted in the picture, the whole project is composed of several parts : mechanical arm, carrier car, photo frame/head mold and camera. We used the reinforcement learning program, such as DDPG and PPO, to learn the angle vector of the next action rotation of the manipulator arm.

The mechanical arm controls the picture frame/head mold and trolley to move to some specific position. The camera collects photos in time and sends it to the face recognition program to judge whether it is a real face or a fake face. The confidence value is returned as a reward, which is a parameter to train the network in the reinforcement learning program. After that, the trained model was tested. The test results were compared with the sampling accuracy.

Confronted with many difficulties, I solve them through my continuous efforts and active discussion with my fellows.

1.Non-convergence: For a certain photo, there is an easy interval for the sample to be breached. However, it is difficult for reinforcement learning to converge, and it is hard to quickly find the breached position for this photo. Therefore, the reward function was modified and some global factors were added to achieve a better convergence.

2.Coordinate system transformation problem in controlling mechanical arm: The mechanical arm cannot be maintained in a fixed position, since matrix multiplication is more complex. There are more than 3 coordinate systems when using the mechanical arm, including geodetic coordinate and flange plate coordinate. After many attempts, I wrote the correct transformation program.

3.Time efficiency problem: Time efficiency of test is inferior to that of sampling. The time waste of data copying and transferring between different CUDA programs in CPU and GPU is solved by unifying all process in GPU.(for example, reinforcement learning is carried out on GPU, but target detection and other parts of the code are CPU version)

4.The lighting problem: Because my superior wanted to make the test more complicated and close to the real situation, I considered adding the lighting system. I had have some experience in the shadow puppet project, so I finally choose ESP8266 WIFI module to control WS2812 8*8 lighting module, which successfully met the requirement.

After this experience in Hikvision pre-research department, I have a deeper understanding of reinforcement learning, mechanical arm control. Also, having a real work experience, I felt the responsibility and burden of my parents.
